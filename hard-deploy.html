--- layout: doc title: hard-deploy description: this task is used to deploy a project to the target VDoc install. permalink: /hard-deploy/ category: doc weight: 2 ---

Full name

com.vdoc.maven:vdoc-maven-plugin:{{site.vdoc_maven_plugin_version}}:hard-deploy

Attributes

  • Requires a Maven project to be executed.
  • The goal is thread-safe and supports parallel builds.
  • Binds by default to the lifecycle phase: package.

Parameters

buildDirectory
Directory containing the generated JAR.
  • java.io.File
  • ${project.build.directory}
dependenciesFolder
where dependencies jar can't be found
  • java.io.File
  • ${project.build.directory}/lib
deployDependencies
custom folder must be updated
  • boolean
  • true
includeSource
source jar must be deployed
  • boolean
  • false
includeTest
test jar must be deployed
  • boolean
  • true
jarName
Name of the generated JAR.
  • java.lang.String
  • ${project.build.finalName}
project
The Maven project.
  • org.apache.maven.project.MavenProject
  • ${project}
session
The Maven session.
  • org.apache.maven.execution.MavenSession
  • ${session}
vdocHome
the VDoc home folder.
  • java.io.File
withCustom
custom folder must be updated
  • boolean
  • true
